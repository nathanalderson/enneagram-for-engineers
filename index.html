Less-Basic Stuff

Closures

A closure is an object consisting of a function and the environment (local variables) in which the function was created. Basically, it's function that remembers its context.

function makeIncrementor(i) {
  return function(x) {
    return x+i;
  }
}

f1 = makeIncrementor(1);
f2 = makeIncrementor(2);
f1(1);	// returns 2
f2(1);	// returns 3

Prototype-based Class System

Class-based (Java) Prototype-based (JavaScript)
Class and instance are distinct entities. All objects are instances.
Define a class with a class definition; instantiate a class with constructor methods. Define and create a set of objects with constructor functions.
Create a single object with the new operator. Same.
Construct an object hierarchy by using class definitions to define subclasses of existing classes. Construct an object hierarchy by assigning an object as the prototype associated with a constructor function.
Inherit properties by following the class chain. Inherit properties by following the prototype chain.
Class definition specifies all properties of all instances of a class. Cannot add properties dynamically at run time. Constructor function or prototype specifies an initial set of properties. Can add or remove properties dynamically to individual objects or to the entire set of objects.
Source: Mozilla Developer Network

Inheritance with Prototypes

C++

class Animal {
public:
  Animal(string name)
    : name(name) {}
  virtual void speak() {}
  string name;
};

class Cat : public Cat {
public:
  Cat(string name, bool cheezburger)
    : Animal(name)

    , canHas(cheezburger) {}
  virtual void speak() {
    if(canHas)
      cout << name << " purrs";
    else
      cout << name << " hisses";
  }
  bool canHas;
};

JavaScript



function Animal(name) {
  this.name = name;
  this.speak = function() {}

}



function Cat(name, cheezburger) {
  this.base = Animal;
  this.base(name);
  this.canHas = cheezburger;
  this.speak = function() {
    if(this.canHas)
      console.log(this.name+" purrs");
    else
      console.log(this.name+" hisses");
  }

}
Cat.prototype = new Animal;

jQuery Deferreds!

What are Promises?

Promises (sometimes called futures) are a way of handling asynchronous code more elegantly than callbacks allow. "Deferred"s are jQuery's implementation of the JavaScript Promises/A API. In jQuery, a Deferred is an object which contains a promise and which can be resolved. A Promise allows users to attach handlers and determine the state.

Many jQuery functions now return Promises by default.

First, let's make a function that returns a promise

$.wait = function(time) {
    var d = $.Deferred();
    setTimeout(d.resolve, time);
    return d.promise();
}

We can use our snazzy new function like this:

$.wait(1000).then(function() {
    console.log("done!");
})

Or, we could save the promise and attach more handlers...

var p = $.wait(1000);
p.then(function() { console.log("I'm done!"); });
p.then(function() { console.log("Me too!"); });

Actually, they're chainable

$.wait(1000)
.then(function() { console.log("I'm done!"); })
.then(function() { console.log("Me too!"); });

But what if everything's not rainbows and unicorns?

$.wait = function(time) {
    var d = $.Deferred();
    if(Math.round(Math.random))
        setTimeout(d.resolve, time);
    else
        setTimeout(d.reject, time);
    return d.promise();
}

We can handle failures easily

$.wait(1000)
.done(function() { console.log("Success! :) ") })
.done(function() { console.log("More success.") })
.fail(function() { console.log("Failure. :( ") })
.always(function() { console.log("The end.") });

My favorite part...

We can combine promises.

var p1 = $.wait(1000);
var p2 = $.wait(2000);

$.when(p1, p2)
.done(function() { console.log("Success! :) ") })
.fail(function() { console.log("Failure. :( ") })

Final notes

  • You can add a handler any time. If the promise has already resolved/rejected, the handler will run immediately as appropriate.
  • There are ways to monitor progress on a deferred.
  • There are ways of modifying the success/fail status of a referred during the handler chain.
